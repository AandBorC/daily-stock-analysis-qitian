---

## 1. 项目定位

基于 AI 大模型的多市场股票智能分析系统，核心流程：

```
抓取数据 → 技术分析/新闻检索 → LLM 分析 → 生成报告 → 通知推送
```

- 支持市场：A股、港股、美股、ETF，日/韩股有限支持
- 推送渠道：企业微信、飞书、Telegram、Discord、Slack、邮件
- 部署方式：GitHub Actions（零成本）、Docker Compose、本地直接运行

---

## 2. 仓库结构（核心）

| 目录/文件 | 作用 |
|----------|------|
| `main.py` | 分析任务主入口 |

## 3. 本地最小运行步骤

### 3.1 准备环境

```bash
# Python 3.10+ 虚拟环境
cd daily_stock_analysis
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
```

### 3.2 安装依赖

```bash
pip install -r requirements.txt
```

> 注意：`requirements.txt` 里有 `git+https://github.com/ZhuLinsen/alphasift.git@...` 这种直接 git 依赖，网络不好可能失败，可暂时注释掉（AlphaSift 是选股增强模块，非必须）。

### 3.3 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env`，最少填写：

```bash
# 自选股（改成你要跟踪的）
STOCK_LIST=002475,002371,002080

# AI 模型（三选一）
# 方式1：DeepSeek（OpenAI 兼容）
OPENAI_API_KEY=sk-xxxxxxxx
OPENAI_BASE_URL=https://api.deepseek.com
OPENAI_MODEL=deepseek-chat

# 方式2：Anspire
# ANSPIRE_API_KEYS=xxx

# 方式3：其他 OpenAI 兼容服务
# OPENAI_API_KEY=xxx
# OPENAI_BASE_URL=xxx
# OPENAI_MODEL=xxx

# 通知渠道（至少配一个）
# 企业微信机器人
WECHAT_WEBHOOK_URL=https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxx

# 飞书机器人
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/xxx
```

### 3.4 运行一次分析

```bash
python main.py --no-notify
```

如果只想看数据是否通，不触发 LLM：

```bash
python main.py --dry-run
```

## 4. GitHub Actions 零成本部署（最推荐）

这是原作者最推荐的方案，适合没有服务器的场景。

### 4.1 配置 Secrets

进入 GitHub 仓库 → `Settings` → `Secrets and variables` → `Actions` → `New repository secret`：

| Secret 名称 | 说明 | 必填 |
|------------|------|:----:|
| `OPENAI_API_KEY` | DeepSeek / OpenAI 兼容 Key | 是 |
| `OPENAI_BASE_URL` | 例如 `https://api.deepseek.com` | 是 |
| `OPENAI_MODEL` | 例如 `deepseek-chat` | 是 |
| `STOCK_LIST` | 自选股代码，逗号分隔 | 是 |
| `WECHAT_WEBHOOK_URL` | 企业微信机器人地址 | 推荐 |
| `FEISHU_WEBHOOK_URL` | 飞书机器人地址 | 推荐 |

### 4.2 触发工作流

`.github/workflows/` 下已包含定时任务，默认收盘后自动运行。也可以手动触发：

`Actions` → 选择工作流 → `Run workflow`

## 5. Docker Compose 部署

适合有服务器、想长期稳定运行的场景。

```bash
# 构建并启动
docker-compose -f ./docker/docker-compose.yml up -d

# 查看日志
docker-compose -f ./docker/docker-compose.yml logs -f
```

Web 界面默认在 `http://服务器IP:8000`。

## 7. 已知注意事项

1. **依赖安装可能报错**：`alphasift` 是 git 依赖，国内网络可能失败，可注释后单独安装。
2. **数据接口稳定性**：efinance/akshare 依赖东方财富等公开接口，接口调整可能导致临时失败。
3. **API Key 成本**：DeepSeek 有免费额度，但大量调用仍需注意预算。
4. **本仓库为学习版**：删除了 `.claude/`、前端 `apps/` 和部分测试/图片资源，保留核心代码、文档和配置。
